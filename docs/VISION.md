# Cuprum — видение и дорожная карта

Cuprum начинался как утилита «засветить Gerber UV-экраном Elegoo Saturn 4 Ultra 16K
для фотолитографии плат». Идея выросла в **CAM-систему для домашнего производства печатных плат**:
один инструмент на весь цикл — от fab-пакета до готовой платы.

## Почему это работает

- **Засветка параллельна.** Весь рисунок экспонируется одной вспышкой за ~90 с —
  независимо от плотности дорожек и числа плат на столе. CNC/лазер работают
  последовательно (время растёт с длиной реза); фотолитография — нет.
- **Тонкий шаг.** 16K-экран ≈ 14×19 мкм/пиксель — дорожки тоньше, чем у дешёвого фрезера.
- **Гибрид по сильным сторонам инструментов:** засветка — дорожки/пятаки, травление —
  убрать медь, **CNC — то, где засветка бессильна: сверловка отверстий и обрезка контура.**

## Текущее состояние (на 2026-05-29)

- **Ядро (`cuprum-core`, Rust):** парс Gerber, растеризация (tiny-skia), композ на
  экран 15120×6230, `.goo`-энкодер, SDCP (discovery/upload/expose). Шаг откалиброван
  (14×19 мкм), коррекция ориентации экрана (поворот 180°), параллельный композ (rayon +
  дедуп), кэш масок.
- **CLI (`cuprum-cli`):** discover / gerber-info / render / calibrate / prepare / upload /
  expose / print.
- **UI (`cuprum-ui`, Tauri 2 + React):** превью с нативной чёткостью, навигация как во
  вьюверах (зум к курсору, пан, CAD-сетка, снап с направляющими), мультивыбор + marquee,
  выравнивание/распределение, авто-раскладка (Arrange/Fill), persist, отправка на засветку.

## Видение: production-CAM

Расширить от «засветки» до полного цеха.

1. **Импорт fab-пакета** (ZIP/папка гербер-джоба из KiCad): авто-классификация слоёв по
   имени — `F_Cu` / `B_Cu` / `Edge_Cuts` / soldermask / silkscreen / drill (`.drl`, Excellon).
2. **3D-превью платы:** стек слоёв (подложка по `Edge_Cuts`, медь, зелёная маска,
   шелкография, отверстия-цилиндры) — напр. react-three-fiber. «Вау»-фактор и контроль сборки.
3. **Процесс = (слой, машина, действие):** выбираешь слой и что с ним делать:
   - `F_Cu` → **UV-засветка** (есть);
   - drill → **CNC-сверловка**;
   - `Edge_Cuts` → **CNC-обрезка / раут**.
4. **Реперы (fiducials)** — один набор крестов связывает все процессы: засветил с реперами →
   CNC выставил ноль по ним → точно попал сверлом/контуром. Делает совмещение
   «засветка → сверловка» и двусторонку автоматическими. (Центр-по-кресту удобнее пинов.)

### Что новое нужно написать

- Парсер **Excellon** (`.drl`).
- Генератор **G-code** для CNC (сверловка/раут).
- **3D** (react-three-fiber).
- Абстракция «процесс» и абстракция **принтера/машины** (разрешение, шаг, протокол).

### MVP-срез (чтобы не утонуть)

ZIP-импорт → список слоёв → 2D-превью (**есть**) → 3D → CNC-экспорт. Каждый шаг полезен сам
по себе.

## Модель проекта

Проект — не «один gerber на экране», а **библиотека плат + панель**:

- несколько fab-пакетов (импортов) в одном `.cuprum`;
- **панель** — композиция экземпляров плат в мм-координатах; от неё идут превью,
  DFM и все процессы (засветка, CNC);
- **пины панели** — привязочные отверстия для переноса между инструментами и
  совмещения верх/низ (отдельно от реперов на меди).

Подробная схема данных, СК и UI-поток — [`docs/PROJECT.md`](PROJECT.md).

## Бэклог (ближайшее)

- **Сохранение проекта** — самодостаточный `.cuprum` (JSON) со **встроенным** гербером
  (не ссылки на пути), чтобы открывался даже без исходников. Нужен вариант рендера превью
  из содержимого, а не из пути.
- **Редактор панели** — раскладка экземпляров import, pins, превью панели (см.
  [`PROJECT.md`](PROJECT.md)).
- **Live-превью в пикселях матрицы** — рендер в нативной анизотропной сетке LCD (14×19 мкм),
  показ реальных зажигаемых пикселей при зуме (nearest-neighbor), live при редактировании.
- **Реперы + регистрация для CNC** (см. видение).
- **Поддержка других принтеров** — профили моделей, абстракция протокола.
- **Экспорт `.goo` и других форматов** из UI (PNG-маска, CTB и пр.) — для других машин /
  ручной загрузки.
- **Оптимизация** по мере необходимости (кэш масок уже есть).
